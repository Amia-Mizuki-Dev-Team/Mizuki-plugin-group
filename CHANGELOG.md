# Changelog

## [Unreleased] - 2026-07-21

### 公告系统收口

- 明确插件职责仅限公告查看、增加、修改、删除、统计、自动补发和发送历史记录。
- 保留最多 5 条公告、单条最多 2000 字符的运行时约束。
- 增强损坏 JSON、错误结构、空白内容、超长内容和越界序号的安全处理。
- 公告文件通过同目录临时文件和原子替换写入；写入失败不会伪造发送成功。
- 发送历史写入会合并最新文件内容，降低并发补发时互相覆盖的风险。
- 使用目标级异步锁避免同一群或私聊目标重复补发；失败后始终释放锁并允许重试。
- 群聊和私聊历史按命名空间隔离，公告内容变化后通过内容哈希重新触发补发。
- 失败 matcher 和公告插件自身不会触发公告补发。
- 公告 matcher 改为不阻断无关消息，使经济、PJSK 和其他插件继续收到同一事件。

### 权限

- 继续支持 NoneBot `SUPERUSER`。
- 支持通过 `AMIA_NOTICE_ADMIN_QQ_IDS` 配置真实 QQ 号；默认包含部署管理员 `2338680148`。
- 在 Gensokyo 虚拟会话 ID 场景下，优先通过 qbind 映射到真实 QQ 号后再判断公告管理权限。
- 权限解析异常默认拒绝。

### 验证

- `python -m unittest discover -s tests -v`：25/25 通过。
- 覆盖公告解析、权限、原子存储、并发历史合并、失败重试、补发去重和 matcher 阻断行为。

### 范围声明

本插件不提供禁言、踢人、管理员设置、群成员管理、群审批、群黑名单或完整群管框架。PermissionProvider 和 AuditLogger 仍是后续对接契约，不在本轮擅自扩展。
